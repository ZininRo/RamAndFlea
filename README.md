# 🐑 Rams and Fleas

Настольная игра в жанре physics-puzzle на Java, вдохновлённая механикой Angry Birds. Запускайте блох из рогатки, разрушайте конструкции и уничтожайте баранов!

---

## Описание

**Rams and Fleas** — однопользовательская игра с физическим движком, реализованная на Java Swing + JBox2D. Цель каждого уровня — уничтожить всех баранов, используя ограниченное количество снарядов-блох. Чем меньше блох потратите, тем больше звёзд получите.

---

## Геймплей

- Зажмите и потяните рогатку, чтобы прицелиться
- Отпустите — блоха летит, сталкивается с блоками и баранами
- Блоки бывают **разрушаемые** и **неразрушаемые**
- **Система звёзд:**
  - ★★★ — 2 или более блох остались после победы
  - ★★ — 1 блоха осталась
  - ★ — все блохи использованы, бараны уничтожены
- Следующий уровень открывается только после прохождения предыдущего

## Архитектура

Проект построен по паттерну **MVC**:

```
src/main/java/org/example/
├── Controller/
│   └── WindowController.java     # главный контроллер, управляет экранами
├── Model/
│   ├── User.java                 # профиль пользователя и прогресс
│   ├── UserManager.java          # регистрация, вход, сохранение в JSON
│   ├── LevelConfig.java          # загрузка уровней из levels.json
│   └── GameObjectDto.java        # DTO игрового объекта
├── Physics/
│   ├── PhysicsWorldManager.java  # создание тел, шаг симуляции, коллизии
│   ├── PhysicsUserData.java      # данные тела: тип, HP, флаги
│   └── PhysicsType.java          # FLEA, RAM, BLOCK, GROUND, WALL, CEILING
└── View/
    ├── BaseWindow.java           # абстрактная панель с общими утилитами
    ├── StartWindow.java          # главное меню
    ├── LoginWindow.java          # вход и регистрация
    ├── GameWindow.java           # выбор уровня со звёздами
    ├── LevelPanel.java           # игровой экран с физикой
    ├── AccountWindow.java        # профиль пользователя
    ├── GameState.java            # перечисление состояний (экранов)
    └── UIUtils.java              # загрузка иконок, создание кнопок
```

## Уровни

Уровни описываются в `src/main/resources/levels.json`. Каждый объект задаётся координатами, размерами, типом (`BLOCK` / `RAM`) и параметрами прочности.

| Уровень | Блох | Описание                                                |
|---------|------|---------------------------------------------------------|
| 1       | 3    | Один баран за двумя блоками, простое введение           |
| 2       | 4    | Два барана, один скрыт за крепостью из блоков           |
| 3       | 5    | Четыре барана на высоких башнях, сложная геометрия      |

Чтобы добавить уровень, достаточно дописать новый объект в массив `levels` в `levels.json`.

## Тесты

```bash
mvn test
```

Покрытие тестами:

- **`UserManagerTest`** — регистрация, вход, обновление прогресса, сброс, удаление, разблокировка уровней
- **`PhysicsTest`** — создание всех типов физических тел, размещение и запуск блохи
- **`ControllerViewTest`** — переключение между экранами через контроллер (с Mockito)

## Данные пользователей

Прогресс сохраняется локально в файл `users.json` в рабочей директории приложения. Файл создаётся автоматически при первой регистрации.

Структура записи:

```json
[
  {
    "username": "player1",
    "password": "pass",
    "levelStars": { "1": 3, "2": 2, "3": 0 }
  }
]
```

> ⚠️ Пароли хранятся в открытом виде — не используйте реальные пароли.
